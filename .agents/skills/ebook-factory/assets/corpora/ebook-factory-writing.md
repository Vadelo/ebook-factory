# Ebook Factory Writing

## Teaching Through Ebooks

Practical ebooks should teach in layers: concept, example, application, reflection, exercise. Readers need to know what to do next, not merely agree with the author.

## Explanation Pattern

For each important idea:

1. Name the idea.
2. Explain it in plain language.
3. Show why it matters.
4. Give an example.
5. Give a small action.
6. Warn about one common mistake.

## Exercises

Good exercises are specific and bounded. Avoid vague prompts like "think about this." Use:

- write a list;
- compare options;
- fill a table;
- audit a current habit;
- draft a plan;
- choose one next action.

## Voice

Use direct, calm, useful prose. Avoid hype. Avoid academic density unless the user requested it. Explain technical terms once, then use them consistently.

## Chapter Openings

Open chapters with the reader's problem or decision, not with generic motivational framing. Avoid repeating "in today's world" across chapters.

## Practical Examples

Examples should include constraints:

- budget;
- time available;
- skill level;
- tools required;
- risks;
- trade-offs.

This makes advice feel real and prevents empty inspiration.
